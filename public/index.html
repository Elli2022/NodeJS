nodejs-microservice-exercises

Production-style Node.js microservice: Express REST API, repository pattern, Neon Postgres in production, filesystem for local tests, Mocha coverage, and progressive refactoring branches.

Node.js Express TypeScript Neon Postgres Render Mocha
Try live API View on GitHub Related Next.js app

Live API: nodejs-microservice-exercises.onrender.com — GET /health returns storage: "postgres" when deployed with Neon. This Netlify page is documentation only.

Render free tier sleeps when idle. The first request after a while can take up to ~60 seconds — that is normal for the demo.

API endpoints

  • GET /health — status + storage backend
  • GET /api/v1/ — list users
  • POST /api/v1/ — create user (validated, hashed password)

Architecture

  • Docs → Netlify (public/)
  • API → Render (Node 20)
  • DB → Neon serverless Postgres
  • Tests → filesystem repo (no DB required in CI)

Try live (curl)

curl https://nodejs-microservice-exercises.onrender.com/health

curl -X POST https://nodejs-microservice-exercises.onrender.com/api/v1/ \
  -H "content-type: application/json" \
  -d '{"username":"demo123","password":"secret123"}'

Run locally

npm install
NODE_HOSTNAME=127.0.0.1 NODE_PORT=3000 NODE_NAME=nodejs-ms npm start

Uses filesystem storage by default. See docs/DEPLOY.md for Neon + Render setup.
